# Guides

Practical, task-oriented walkthroughs for adopting `oz`.

## Start here

1. [`first-workspace.md`](./first-workspace.md) — scaffold and understand a new workspace.
2. [`graph-and-mcp.md`](./graph-and-mcp.md) — use graph query and MCP in daily flow.
3. [`adopt-existing-repo.md`](./adopt-existing-repo.md) — introduce `oz` in an existing repository.
4. [`first-agent-and-skill.md`](./first-agent-and-skill.md) — customize routing and behavior.
5. [`team-rollout.md`](./team-rollout.md) — optional guidance for team-level rollout.

## Guide format

Each guide follows the same shape:

- Goal
- Preconditions
- Steps
- Verify
- Common pitfalls

If guidance conflicts with convention details, treat `specs/oz-project-specification.md` as canonical.
