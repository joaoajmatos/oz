# Guides

Practical, task-oriented walkthroughs for adopting `oz` as a shared workspace environment.

## Start here

1. [`first-workspace.md`](./first-workspace.md) — scaffold and understand a new workspace.
2. [`graph-and-mcp.md`](./graph-and-mcp.md) — use graph query and MCP in daily flow.
3. [`context-tuning.md`](./context-tuning.md) — tune routing and retrieval with `oz context scoring`.
4. [`adopt-existing-repo.md`](./adopt-existing-repo.md) — introduce `oz` in an existing repository.
5. [`first-agent-and-skill.md`](./first-agent-and-skill.md) — customize routing and behavior.
6. [`ops-and-recovery.md`](./ops-and-recovery.md) — day-2 health checks and safe recovery with `oz repair`.
7. [`crystallize-notes.md`](./crystallize-notes.md) — classify notes and review promotion-ready diffs.
8. [`team-rollout.md`](./team-rollout.md) — optional guidance for team-level rollout.

Use these guides whether your team works in one repository or a meta-repo model with separate codebases connected as git submodules under `code/`.

## Guide format

Each guide follows the same shape:

- Goal
- Preconditions
- Steps
- Verify
- Common pitfalls

If guidance conflicts with convention details, treat `specs/oz-project-specification.md` as canonical.
