Historical planning artifacts for shipped `oz` features live here: the original PRDs, pre-mortems, and sprint plans that shaped the work, but are no longer the normative description of the current system. In the oz trust hierarchy, this material sits in `notes/` (working input) rather than `docs/` (stable reference). The shipped contracts and behaviour now live in `specs/` and `docs/architecture.md`; use this folder when you want design history, rollout context, or the reasoning that led to what is implemented today.
